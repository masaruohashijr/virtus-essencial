E3.22. Consulta ARC para usuários designados

Pré-requisitos

Deve existir ARCs designados para o usuário logado no sistema.

Perfil de usuário

Qualquer perfil válido no sistema

Item de menu

Tela inicial da aplicação para o perfil inspetor

Telas

Título: Painel do Inspetor

Trilha: Início --> Painel do Inspetor

Código de tela: [APSFW0601]

Comando: Selecionar ARC.

Descrição

Funcionalidade básica de listar os ARCs designados para o usuário logado.

Tela inicial disponível apenas para o perfil Inspetor.

Regras específicas

Detalhamento:
  • Tela inicial da aplicação para o inspetor.
  • A tela inicial da aplicação deve exibir a relação dos ARCs designados para o inspetor logado no sistema. ARCs nos estados:
    • 1 - Previsto.
    • 2 - Designado.
    • 3 - Em edição.
    • 4 - Preenchido.
    • 5 - Análise delegada.
    • 6 - Em análise.
  • Esta tela permite a seleção do ARC para:
    • detalhar, incluir e alterar notas do ARC - se o ARC estiver nos estados 2 - Designado ou 3 - Em edição e somente para o usuário que foi designado para atualizar o ARC.
    • detalhar, analisar (aprovar/rejeitar) ARC - se o ARC estiver nos estados 4 - Preenchido ou 5 - Análise delegada e somente para o usuário que foi delegada a analise do ARC.
    • detalhar - se o ARC estiver em qualquer estado diferente dos demais somente para o usuário que foi designado/delegado para o ARC.
  • As funcionalidades básicas de um usuário com perfil inspetor são detalhar, incluir e alterar ARCs para ele designado.
  • No entanto, o supervisor responsável pela ES poderá delegar a um outro usuário a funcionalidade que permite analisar o ARC.
  • Ciclo de vida do ARC:
    • 1 - Previsto: na criação do ARC após liberação da matriz na inclusão manual do ciclo ou na inclusão automática do ciclo, ambos por usuário com perfil supervisor (responsável da ES).
    • 2 - Designado: após a designação de atualização do ARC pelo usuário com perfil supervisor (responsável da ES).
    • 3 - Em edição: após alteração do ARC pelo usuário designado. Permanece neste estado até o usuário designado concluir o preenchimento.
    • 4 - Preenchido: usuário informa que finalizou a inclusão de notas.
    • 5 - Análise delegada: supervisor responsável da ES delega a análise do ARC para outro usuário.
    • 6 - Em análise: supervisor responsável da ES ou outro usuário por ele delegado está analisando o ARC.

Cenários de teste

Dado que:

  • ARCs nos estados 1 - Previsto; 2 - Designado; 3 - Em edição; 4 - Preenchido; 5 - Análise delegada; 6 - Em análise.
  • ARCs designados para dois usuários distintos com perfil inspetor.

Observação: agrupar a maior quantidade de itens de teste no mesmo teste.

Negócio.

  • C001 Falha - Usuário logado não possui ARCs designados. Mensagem:
    • "Não há ARCs designados."
  • C002 Sucesso - Apresenta a relação de ARCs designados para o usuário logado

Web.

  • Teste web equivalente - C001 - Conferir apenas a mesangem.
  • Teste web equivalente - C002 - Conferir a listagem de ARCs.
  • Conferir se o botão Selecionar ARC está habilitado somente para ARCs selecionados nos estados 2 - Designado ou 3 - Em edição.
  • Sucesso - Testar a navegabilidade do comando Selecionar ARC que deve direcionar o usuário para a tela de edição de ARC.
